# 项目介绍与启动说明

## 项目简介

`AI-opportunity-score` 是一个面向政企场景的 AI 商机发现与研判项目。

这个项目的目标不是单纯搜索“AI”关键词，而是从公开网页、政策通知、招采公告、组织动态等信息中，发现值得跟进的政企 AI 改造机会，并输出结构化研判结果。

系统核心能力包括：

- 通过关键词触发公开信息检索
- 对线索做去重、分类和结构化入库
- 用 Agent 对线索进行初筛和深查
- 对线索输出评分、理由、建议动作和时效状态
- 在前端看板中展示线索列表、详情和任务结果

## 核心链路

项目当前主要围绕这条链路运行：

1. 配置关键词
2. 手动触发采集
3. 检索公开网页并入库
4. Agent 对线索做初筛
5. 对高价值线索做深查
6. 后端聚合评分结果
7. 前端展示线索看板和详情

## 项目结构

项目主要分为 3 部分：

- `ai-opportunity-frontend`
  - 前端页面，技术栈是 `Vue + Vite`
- `ai-opportunity-backend`
  - 后端服务，技术栈是 `Spring Boot`
- `ai-opportunity-agent`
  - Agent 执行模块，负责检索、初筛、深查和结果输出

## 启动前准备

本项目本地运行建议先准备：

- `Node.js`
- `JDK 17`
- `MySQL`
- `ai-opportunity-agent/.env` 中的模型和检索配置

数据库默认连接配置见后端 `application.properties`，默认是：

- host: `127.0.0.1`
- port: `3306`
- database: `ai_opportunity_score`
- username: `root`
- password: `root`

## 启动项目

### 1. 启动数据库

如果使用 Docker：

```bash
docker run -d --name ai-opportunity-mysql \
  -e MYSQL_ROOT_PASSWORD=root \
  -e MYSQL_DATABASE=ai_opportunity_score \
  -p 3306:3306 \
  mysql:8.4
```

如果你已经导入过数据库数据，确认数据库正常运行即可。

### 2. 启动后端

进入目录：

```bash
cd ai-opportunity-backend
```

启动命令：

```bash
.\mvnw.cmd spring-boot:run
```

### 3. 启动前端

进入目录：

```bash
cd ai-opportunity-frontend
```

首次运行先安装依赖：

```bash
npm install
```

启动命令：

```bash
npm run dev
```

### 4. 打开项目

前端默认访问地址：

```text
http://localhost:5173
```

后端默认端口：

```text
http://localhost:8080
```

## 说明

- 前端通过 `/api` 调用后端接口
- Agent 由后端按任务触发，不是单独的 Web 服务
- 如果需要执行真实检索和分析，必须保证 `ai-opportunity-agent/.env` 配置完整
- 如果只是演示已有数据，优先保证数据库和后端能正常启动
